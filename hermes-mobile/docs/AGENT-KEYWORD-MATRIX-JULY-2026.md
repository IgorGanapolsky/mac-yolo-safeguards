# Hermes Mobile AI-agent keyword matrix (July 2026)

## Positioning

Hermes Mobile is the independent phone control plane for a Hermes gateway the user operates.
It does not sign in to or replace Cursor, Claude Code, ChatGPT, Codex, Gemini, Replit,
Copilot, Windsurf, OpenClaw, or their mobile products.

The discovery strategy targets the operator problem shared by those audiences while keeping the
integration boundary explicit. This avoids ranking copy that converts the wrong user and creates a
bad review after installation.

## Metadata implemented

- Google Play title: `Hermes Mobile: AI Agent Leash` (29/30 characters).
- Google Play description: names all major agent ecosystems once, explains the non-integration
  boundary, and separates free chat/state from paid Leash approval.
- App Store name: `Hermes Mobile — AI Control` (existing broad category signal).
- App Store subtitle: `AI Coding Agent Control` (23/30 characters).
- App Store keywords: 92/100 characters, comma-separated and focused on competitor/category terms
  not already covered by the name and subtitle.

## Content rotation

The twice-daily social engine rotates one ecosystem term per run and includes the wider set in
long-form content. Short posts use the pattern:

> `[Agent] users know this operator problem. Hermes Mobile controls your own Hermes gateway; it
> does not log in to or replace [Agent].`

This creates relevant indexed language without false compatibility claims or repeated keyword
blocks that read as spam.

## Review and paid-upgrade path

1. Search or social post drives the Google Play visit.
2. The user installs and pairs a real Hermes gateway.
3. A successful first run can earn an honest review.
4. A failed first run routes to the exact setup step, not review manipulation.
5. Users who need remote command decisions evaluate Leash Pro after chat/state activation.

## Source constraints

- Apple says app search considers the app name, subtitle, keywords, and primary category:
  <https://developer.apple.com/app-store/search/>.
- Google Play supports keyword-targeted custom store listings, which can later split Cursor,
  Claude Code, Codex, Gemini, and Replit acquisition traffic:
  <https://support.google.com/googleplay/android-developer/answer/9867158>.
- Replit is treated as a different job, not a replacement claim; current comparison evidence is in
  [COMPETITIVE-REPLIT-AGENT.md](./COMPETITIVE-REPLIT-AGENT.md).

Store ranking is an outcome, not a metadata completion claim. Upload status, indexation, impressions,
store conversion, retained activation, reviews, and Leash purchases need separate evidence.
