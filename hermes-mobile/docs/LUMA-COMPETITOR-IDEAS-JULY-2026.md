## Top 5 stealable ideas — Hermes-specific implementation

### 1. Honest critic positioning (marketing)

**Steal:** "Stop AI from just agreeing with you — make it an honest critic."

**Hermes version:** "Your coding agent wants to run `git push --force`. Leash is the honest critic that makes it ask first."

**Implement:**
- Social: [docs/social/week-2026-07-10/06-oneday-honest-critic-angle.md](./social/week-2026-07-10/06-oneday-honest-critic-angle.md)
- Play A/B variant D: `fastlane/metadata/android/en-US/variants/short_description_D_honest_critic.txt`
- Do **not** change iOS metadata until review completes

### 2. Walk-away-with bullet structure (ASO + social)

**Steal:** Three concrete outcomes above the fold.

**Hermes version:**

> **What you get in 10 minutes**
> - QR-pair your Mac — keys never leave your hardware
> - Approve or deny the next risky tool call from your phone
> - A repeatable safety gate for Cursor, Claude Code, and Hermes gateway

**Implement:** Added to social draft; optional Play full desc prepend after A/B baseline established.

### 3. Free live workshop on Luma (community / GTM)

**Steal:** 366-person free Zoom funnel via Luma Discover.

**Hermes version:** "Pressure-test your Cursor setup — live Leash approve/deny demo" · 30 min · Play + App Store links · monthly.

**Implement:** plan.md §4 discovered task; event page template in social draft folder; Igor hosts (no eng).

### 4. Pressure-test onboarding (product)

**Steal:** "Apply it live to your own idea."

**Hermes version:** After first pair, gateway sends a **sandbox** `echo` or read-only tool call → user must approve in Leash → confetti + "Your gate works."

**Implement:** plan.md §4 — requires claim on `FreshUserOnboardingCard` / gateway sandbox endpoint.

### 5. ICP pain bullets in connection UI (product UX)

**Steal:** Four "why this is for you" bullets targeting pre-launch founders.

**Hermes version** (for `ConnectMacGate` / disconnected state):

- You run Cursor or Claude Code but step away from your desk
- You've seen an agent run a command you wouldn't have approved
- You don't want another cloud relay reading your repo
- You need one tap to approve or block — not an SSH terminal

**Implement:** plan.md §4 — copy-only change in onboarding components (needs claim + fresh-user tests).
