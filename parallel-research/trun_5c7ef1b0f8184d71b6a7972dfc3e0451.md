# Hermes Mobile: Stay on GitHub or Move to GitLab for Agentic Development?
**Decision date:** July 13, 2026
**Audience:** CTO, executive sponsors, mobile platform lead
**Scope:** mac-yolo-safeguards/hermes-mobile (Expo / React Native monorepo)

## Executive Recommendation
**Recommendation: Option A - stay on GitHub and harden orchestration.** Run a parallel 2-week GitLab Mirror Sandbox to validate whether Duo Agent Platform flows materially outperform the existing Copilot Coding Agent workflow on a non-critical surface, and re-decide in Q1 2027. Migrating now is not justified: EAS Workflows has a first-class GitHub Actions integration but no equivalent GitLab CI integration, GitLab SaaS macOS runners remain beta-gated, and agentic parity between the two platforms in mid-2026 is closer than the marketing suggests.

The decisive facts as of July 13, 2026:
- GitLab Duo Agent Platform reached GA in GitLab 18.8 (January 2026) and requires the Duo Pro ($19/seat/mo) or Duo Enterprise add-on ([32]; [129]).
- GitHub Copilot Coding Agent reached GA across all paid Copilot plans on February 21, 2026 (GitHub blog "Meet the new coding agent"; [43]).
- GitLab SaaS macOS runners are still in Beta, Premium/Ultimate-only, with 3-hour job timeouts and Apple-silicon only ([19]).
- EAS Workflows' first-party CI integration is documented for GitHub Actions, not for GitLab CI (Expo, How to integrate EAS Workflows with GitHub Actions).
- GitHub Actions hosted macOS runners (M1/Intel larger runners) are GA at $0.062/min (GitHub Actions runner pricing; [6]).

## State of the Two Agentic Platforms (mid-2026)

### GitLab Duo Agent Platform
Reached GA in GitLab 18.8 (January 2026). The platform orchestrates specialised agents (Duo Planner, Duo Agentic Chat, Software Development Flow) against issues, MRs, and vulnerabilities, driven by `AGENTS.md` context files ([100]; [101]). Licensing requires the Duo Pro or Duo Enterprise add-on; Duo Basic is bundled with Premium/Ultimate but excludes the Agent Platform ([129]).

Hosted macOS runners remain a meaningful constraint for an Expo / React Native shop: GitLab SaaS macOS runners are in Beta, gated to Premium and Ultimate tiers, with `saas-macos-medium-m1` (4 vCPU / 8 GB) and `saas-macos-large-m2pro` (6 vCPU / 16 GB) images and a 3-hour per-job ceiling ([19]).

### GitHub Copilot Coding Agent + Agent Mode + Agents HQ
Copilot Coding Agent reached GA across all paid plans on Feb 21, 2026. Assigning an Issue to Copilot triggers a background Actions runner that produces a draft PR (GitHub blog, "Meet the new coding agent"). Agents HQ, announced the same window, routes work between Copilot, Claude, and Codex inside one repo (GitHub blog "Agents HQ"). MCP support is uniform across IDE, CLI, the cloud agent, and code review surfaces (GitHub MCP docs).

### Hermes-specific tool reality
- **EAS Workflows**: first-party GitHub Actions integration is documented; no equivalent GitLab CI integration is documented (Expo blog).
- **Dependabot**: native to GitHub; GitLab's equivalent is Dependency Scanning, which is an Ultimate-only feature and lacks Dependabot's auto-PR ergonomics for npm/CocoaPods/Gradle.
- **Sentry/PostHog**: both ship official integrations for GitHub and GitLab. Sentry's GitLab integration uses the generic repo-webhook model; the GitHub one is the reference implementation and benefits from tighter permission scoping ([61]; Sentry GitHub integration).
- **macOS builds**: GitHub Actions hosts M1 and Intel macOS runners as GA, including larger runners with longer timeouts ([6]).

## Where GitLab Actually Wins for Hermes

1. **One platform = repo + CI + Duo + security + runners.** Hermes currently stitches GitHub + EAS + Sentry + PostHog + Dependabot + macOS larger runners. GitLab Ultimate consolidates these, reducing the number of consoles, secrets stores, and SSO integrations from roughly five to one.
2. **Merge trains** (Premium/Ultimate) solve the "fast-forward while CI is queued" problem that emerges when three feature branches race to `main` near a release.
3. **Built-in SAST/DAST/dependency/container scanning** on Ultimate removes the need for an external security tool.
4. **Self-managed Ultimate** exists, which matters if Hermes ever needs FedRAMP or on-prem for a regulated customer, given the `mac-yolo-safeguards` namespace hints at a safety/AI-compliance posture.
5. **Duo Agent Platform flows** are a more opinionated multi-agent pipeline; for a team without an existing `plan.md` + `worktrees` convention, GitLab's defaults may be a faster onboarding than assembling one on GitHub.

## Where GitHub Actually Wins for Hermes

1. **EAS Workflows / Expo Application Services.** EAS is the release spine and is built and documented against GitHub Actions. Migrating forces rewriting CI to call `eas-cli` from GitLab CI without first-party support for `eas workflow:` triggers (Expo blog).
2. **Maestro device-farm E2E.** Maestro's CI templates target GitHub Actions; on GitLab you hand-roll runner bring-up.
3. **Dependabot.** No equivalent UX on GitLab.
4. **Codespaces.** No first-party equivalent on GitLab; GitLab Workspaces is newer and less mature for React Native monorepos.
5. **macOS runner availability.** GitHub Actions M1 and Intel images are GA with larger runners and 7-hour timeouts; GitLab SaaS macOS is beta and capped at 3 hours (GitHub runner pricing; [19]).
6. **Ecosystem and discoverability.** Public Expo / React Native orgs, blog posts, and contributor examples overwhelmingly assume GitHub; contributor friction for external PRs is real.
7. **Copilot Coding Agent + Agents HQ** is the broadest multi-agent surface in mid-2026; GitLab's flows are narrower but tighter.

## Migration Cost and Risk Inventory

| Surface | Effort | Risk |
|---|---|---|
| Repo import (code, branches, tags, LFS) | Hours | Low - GitLab importer handles most of it |
| Issues/PRs/MRs/comments | 1-2 engineer-weeks | Medium - labels map 1:1; Projects v2 boards, issue types, saved replies do not import cleanly |
| GitHub Actions YAML -> `.gitlab-ci.yml` | 2-3 weeks | **High** - different trigger model, no `uses:` marketplace; must port `actions/checkout`, `setup-node`, cache, and secrets handling |
| **EAS Workflows** | 1 week + ongoing | **High** - first-party GitHub integration replaced with manual `eas workflow:run` invocations; OTA promotion paths break |
| Secrets & variables | 1 week | Medium - CI secrets map; `ACTIONS_*` env vars do not; Codespaces secrets must be rebuilt as Workspaces |
| Dependabot configs -> Renovate or Dependency Scanning | 1 week | Medium - Dependabot config does not import; behaviour differs for npm/CocoaPods/Gradle |
| GitHub Apps (Sentry, PostHog, Codecov, etc.) -> GitLab OAuth | 1-2 weeks | Medium - auth flows, webhook payloads, and issue-link semantics differ; some apps lack GitLab OAuth |
| Branch protection / CODEOWNERS | Hours | Low |
| GitHub Pages (docs site) | 1 week | Medium - different build pipeline, custom domains, HTTPS provisioning |
| **Mobile CI cost delta** | TBD | **High** - GitLab macOS runners are beta, 3h timeout, Apple-silicon only on Premium/Ultimate; EAS Build on Expo's own M-series fleet may still be the most cost-effective path |
| URL redirects | 1 week | Medium - 301 redirects from `github.com/...` to `gitlab.com/...` lose SEO equity and break bookmarked links |
| Community / discoverability | Ongoing | **High** - for an OSS or branded project, GitHub is where ML/AI safety talent searches |
| Contributor onboarding re-write | 1-2 weeks | Medium - CONTRIBUTING.md, issue templates, PR templates, Codespaces devcontainer |

**Rollback plan.** Because Git is distributed, a migration is reversible at the code level. The hard-to-reverse parts are: (1) CI pipelines re-written around GitLab YAML, (2) third-party OAuth installed against the GitLab project, (3) community/contributor momentum drained during the move, and (4) Pages redirect chain.

## Three Options Scored

### Option A - Stay on GitHub; harden orchestration
Adopt Copilot Coding Agent in production on feature work with a label-gated workflow. Standardise `plan.md` + worktree conventions in repo and wire them into Agent Mode. Add MCP servers for Sentry, PostHog, and Expo docs. Keep Dependabot, Codespaces, GitHub Actions, and the EAS Workflows integration.

### Option B - GitLab-primary with GitHub read-only mirror
New GitLab project becomes canonical; GitHub repo kept read-only. Duo Pro/Enterprise add-on for the platform team; macOS runners reserved for iOS builds. Re-author EAS Workflows invocation as `eas workflow:run` from `.gitlab-ci.yml`. Use GitLab's `import/github` to migrate issues, MRs, comments, milestones. Cost: GitLab Premium ($29/seat/mo) + Duo Pro ($19) + GitHub Team ($4 mirror).

### Option C - Full cutover to GitLab
Same as B but the GitHub repo is archived; all integrations re-pointed. Highest upfront cost and irreversible community effects.

## Two-Week Reversible Pilot: "GitLab Mirror Sandbox"

**Goal:** validate whether Duo Agent Platform flows materially outperform the existing Copilot Coding Agent workflow on a non-critical surface, without breaking the canonical pipeline.

| Day | Workstream | Owner | Deliverable |
|---|---|---|---|
| 1-2 | Spin up `gitlab.com/free` namespace; create project `hermes-mobile-sandbox`. Enable Duo Pro trial. | DevOps | Sandbox repo + Duo license |
| 2-3 | Mirror the GitHub repo (`git push --mirror`) and a `.gitlab-ci.yml` shim that calls `eas-cli` for builds. | DevOps | Green macOS beta build via GitLab SaaS macOS runner |
| 4-5 | Author `AGENTS.md` describing Hermes conventions; map Duo Planner + Duo Agentic Chat to current `plan.md` template. | Mobile lead | Two agent runs captured |
| 5-7 | Run the same task (small, isolated: a docs refactor + a test fixture) through both Copilot Coding Agent on `main` and Duo Agent Platform on `sandbox`. Record PR cycle time, reviewer comments, and CI signal. | 2 engineers | Side-by-side table |
| 8-9 | Measure macOS runner queue time vs GitHub Actions M1 runner; record build minutes cost. | DevOps | Cost & latency table |
| 10-12 | Toggle one non-customer-facing workflow (e.g. nightly dependency bump) to GitLab CI to validate EAS + GitLab glue. | DevOps | Working nightly job |
| 13 | Decide: extend, expand, or roll back. | CTO | Go / no-go memo |

### Exit Criteria (abort triggers - any one is sufficient)
- macOS beta runner queue time exceeds 2x the current GitHub Actions M1 wait for 3 consecutive runs.
- EAS Workflows integration on GitLab CI requires more than 4 engineering-hours per missed trigger pattern.
- Duo Agent Platform PR cycle time on the pilot task is not within +/-15% of Copilot Coding Agent on the same task.
- Any data residency / IP issue surfaces that requires a self-managed GitLab deployment (would force a 6+ month commitment decision).

### Reversibility
All code lives in Git; the sandbox repo can be deleted at the end of week 2 with zero blast radius to `main`. No production secrets are moved; CI runs against a read-only Expo account. Documentation of lessons learned is stored in `main` regardless of decision.

---

## Final Recommendation
**Adopt Option A now, run the GitLab Mirror Sandbox in parallel, and re-decide in Q1 2027.**

The decisive arguments are:

1. **Mobile CI is the project's spine.** GitHub Actions macOS runners are GA, fast, and the only path EAS Workflows supports natively. GitLab SaaS macOS runners are still beta-gated and 3-hour timeout, which is a poor fit for Expo's iOS archive workflows.
2. **Agentic parity, not agentic lead.** Both platforms have GA multi-agent surfaces in 2026. The market has not yet produced evidence that one reduces PR cycle time materially more than the other for an Expo monorepo. Migrating to chase a marketing claim is a category error.
3. **Migration cost is concrete and non-reversible in community effects.** Even with `import/github` mapping MRs and issues cleanly, you lose Pages discoverability, contributor inbound, and a year of OSS convention around `.github/`.
4. **The optionality is real.** A two-week mirror pilot costs roughly one engineer-week and preserves a clean GitLab path if Q1 2027 benchmarks tip the other way. It is the lowest-regret option.

If, after the pilot, Duo Agent Platform demonstrably improves cycle time by more than 15% on the chosen tasks *and* the EAS-on-GitLab glue is sustainable, re-open Option B as a Q2 2027 decision. Until then, stay on GitHub.

## Synthesis: Comparative Anatomy

| Dimension | GitHub (mid-2026) | GitLab.com (mid-2026) | Verdict for Hermes |
|---|---|---|---|
| Coding agent GA status | Coding Agent GA Feb 2026 across all paid plans | Duo Agent Platform GA Jan 2026 (18.8), Duo Pro/Enterprise add-on required | Tie |
| MCP support | First-class across IDE, CLI, cloud agents, code review | Supported via Duo Agent Platform integrations | Tie |
| Multi-agent orchestration | Agents HQ (Copilot + Claude + Codex in parallel) | "Flows" with Planner, Chat, Software Dev Flow | Slight GitLab (opinionated), slight GitHub (breadth) |
| macOS CI runners (mobile) | GA, M1/Intel, larger runners, 7h timeout | Beta on Premium/Ultimate, M1/M2 Pro, 3h timeout | **GitHub** |
| EAS Workflows | First-class GitHub Actions integration | No first-party integration; manual CLI invocation | **GitHub** |
| Dependabot equivalent | Dependabot native | Dependency Scanning (Ultimate) + Renovate | **GitHub** for UX |
| Codespaces / dev containers | Codespaces GA | Workspaces GA but less mature | **GitHub** |
| Merge queue vs trains | Merge queue GA | Merge trains in Premium/Ultimate | Tie |
| Self-managed option | GitHub Enterprise Server (GHES) | GitLab Self-Managed + Dedicated | Tie |
| Pricing (per seat/mo, hosted) | Team $4 / Enterprise $21; Copilot Business $19 / Enterprise $39 | Premium $29 / Ultimate custom; Duo Pro $19 add-on | **GitHub cheaper** unless consolidating stacks |
| SAST/DAST/container scanning | Code scanning + Advanced Security (Enterprise) | Built-in Ultimate tier | **GitLab** (consolidated) |
| Discoverability / SEO for OSS | Highest | Mid | **GitHub** |
| Migration cost to reverse | 2-3 weeks for repo + MR/issue mapping | Similar | Tie |

**Key insight.** GitLab's agentic story is differentiated but narrow: it is strongest when you want one MR-scoped loop and you can put the model behind a paid add-on. GitHub's story is broader: coding agent, MCP, Agent HQ, and Codespaces form a richer surface, but the multi-agent orchestration is newer and the UX is less opinionated. For Hermes specifically - a mobile monorepo with EAS-first CI - the EAS integration and the macOS runner GA status dominate the comparison and tilt it firmly toward GitHub today.
