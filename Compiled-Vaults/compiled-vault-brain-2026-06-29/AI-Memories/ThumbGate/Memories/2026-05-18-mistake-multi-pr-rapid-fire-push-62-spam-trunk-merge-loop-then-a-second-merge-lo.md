# MISTAKE: Multi-PR rapid-fire push, 62-spam /trunk merge loop, then a second merge loop, all distracting from the actual...

What went wrong: Multi-PR rapid-fire push, 62-spam /trunk merge loop, then a second merge loop, all distracting from the actual revenue-shipping work (Stripe branding). When asked 'are you lost?' I admitted yes — that's a failure mode, not a planning step.
How to avoid: ONE critical-path item at a time. Bias toward ship-today paths (push-triggered workflows, admin merges) over wait-for-queue paths. Never start a second monitoring loop. When pushing to >2 PRs in a session, stop and ask what's actually moving revenue.
